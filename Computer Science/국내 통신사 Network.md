## Network
```
데이터가 이동할 수 있게 연결된 상황 그 자체를 네트워크라고 함
내가 이메일을 보내는것, 카톡, 전화 등등 전부 네트워크가 성립하기 때문에 가능함.

일단 네트워크는 모뎀 > 공유기 > 전자기기

이렇게 연결된다.

통신사에서 개인 주택에다가 모뎀을 설치한다.(이떄 통신사에서 개인주택까지 전봇대가 다리역할을 함)

모뎀은 통신사에서 데이터를 전달할 수 있는 일종의 공인 인증 같은 역할을 한다.

그래서 모뎀이 공인 아이피(고정아이피)를 갖게 된다.

모뎀이 없으면 인터넷 못함

근데 데이터를 받는 기계라면 무조건 각각 주소가 있어야한다.

왜냐면 IP(Internet Protocol) 자체가 인터넷 주소를 의미한다.

핸드폰, 노트북이 몇대이든 IP를 할당받아야 와이파이를 쓸 수 있다.

그래야 데이터가 와이파이를 통해 어디로 갈지 도착지가 정해지니까.

그 역할을 공유기가 한다.

일반적으로 공유기는 모뎀과 랜선으로 연결되어 있는데

공유기(안테나 달린놈)가 다른 기기들에게 알아서 IP를 할당해준다.

이 IP는 수시로 변하고 IP 번호가 비슷비슷하다.

이때 공유기가 주는 IP를 사설 아이피라고 한다.(유동 아이피)


통신사 > 모뎀(공인아이피) > 공유기(와이파이)> 전자기기



```

## KT
```
통신사가 모뎀에다 공인 아이피를 할당하고 다시 모뎀이 공유기에게 공인아이피를 할당해준다.
```

## SKT
``
모뎀이 공인아이피를 할당 받는것까진 똑같은데 공유기는 사설 아이피를 할당받아서 CMD창의 ipconfig의 ip는 사설아이피로 나온다.
``

## 이걸 작성한 이유
```
만약 서버를 연결하고 싶을때 통신사를 SKT를 쓴다면 포기해라

호스팅하는 방법 외엔 없다.

아니 다른 방법이 있긴한데 호스팅하는게 편하다.

공인아이피는 원칙적으로 집집마다 1개만 할당된다.

이 공인 아이피라는건 내 진짜 주소인데 보통 공인아이피를 도메인으로 가려서 남들에게 도메인을 공유헤서 서버에 접속하는 방식이다.

근데 skt는 공유기에서 이미 사설 아이피를 할당하기 때문에 외부에서 들어올 수 없다.

내가 이 사실을 몰랐을때 고객센터에 전화해봤는데 처음엔 장애문의 아니면 안받는다 이지랄해서 절차대로 구라치고 장애문의 넣고 상담했다.

근데 자기들은 잘못이 없다. 제대로된 원인파악은 안해주고 내가 원인을 파악해야한다는 KT사건처럼 씨부리길래 에휴 내가 찾겠다하고

얻어낸 결과이다.

국내 통신사는 병신들이다.

같은 병신들이라면 공인아이피 후하게 할당해주는 KT를 쓰자.
```


## 사설 아이피를 공인아이피로 바꾸면 되잖아
```
그 방법을 이미 해봤다.

공인 아이피로 바꾸는 순간 공인 아이피는 1개의 기기만 이용가능해져서

와이파이는 뜨는데 1개의 기기빼고 다른 기기가 연결이 되지 않는 현상이 생긴다.

공인아이피는 여러 곳에 할당하지 못하기 때문이다.(정해진거니까 외우면됨)

그러면 여기서 다른 방법으로 해결해야 되는데 너무 피곤하다.

웃긴건 리눅스에선 서버연결이 되었다.
```
